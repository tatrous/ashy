# Воркшоп «Продукт за день» — раздатка

BHSAD · Vibecoding Workshops. За один день командой 3–6 человек: из ваших исследований и брифов → книга продукта → фронт + бэк → деплой в интернет.

## Что в папке

| Файл | Что это | Когда нужен |
|------|---------|-------------|
| `workshop.md` | Плейбук: закидываете в Cursor, агент ведёт команду через весь день | Режим А — рекомендуется |
| `prompts.md` | Те же шаги отдельными промптами с метками ролей | Режим Б — ручной контроль |
| `Product_Book_Template.md` | Полный шаблон книги продукта V3 с курса | Опционально: агент возьмёт из него точные таблицы |
| `README.md` | Этот файл | старт |

## Что нужно до старта

- Установленный **Cursor** (cursor.com) с рабочим AI-чатом
- **Node.js LTS** (nodejs.org) и **git** (`git --version` в терминале)
- Почта у каждого в команде — аккаунты GitHub / Supabase / Cloudflare регистрируем прямо на воркшопе, SMS не понадобится
- Windows: PowerShell 7+ или Git Bash, **не CMD**
- Ваши материалы: брифы, исследования, интервью, конкуренты — любые форматы

## Быстрый старт (режим А)

```
mkdir team-mvp
cd team-mvp
git init
mkdir -p docs/research
```

1. Скопируйте `workshop.md` (и по желанию `Product_Book_Template.md`, `prompts.md`) в корень `team-mvp/`.
2. Сложите все материалы команды в `docs/research/`.
3. Откройте папку в Cursor (`cursor .`), чат ⌘L / Ctrl+L, режим **Agent**.
4. Первое сообщение:

```
Прочитай @workshop.md и веди нашу команду по нему. Старт.
```

Дальше агент спросит про команду и материалы, соберёт книгу продукта, поможет с регистрациями, построит продукт и задеплоит.

## Режим Б

Тот же маршрут по шагам: откройте `prompts.md` и копируйте промпты по порядку. У каждого промпта метки — кто из команды запускает и какая модель (код → Sonnet, тексты → Opus).

## Результат дня

- Публичная ссылка на работающий продукт (Cloudflare Pages — базовая рекомендация; Vercel из РФ не регистрируется — нужен SMS)
- `product_book.md` — книга продукта в репозитории
- GitHub-репо с коммитами всей команды
- Пройденный тест изоляции данных (RLS)

## Если что-то сломалось

Аварийные протоколы — в конце `workshop.md` и `prompts.md` (Блок 6): зациклился Cursor, упал деплой, потёк RLS, не цепляется Expo. Совсем всё горит с хостингом — подойдите к Владу, задеплоим на его сервер.
