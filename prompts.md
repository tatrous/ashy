## БЛОК 0 — ПОДГОТОВКА ОКРУЖЕНИЯ (руками, 10 мин)

**Терминал.** Mac: Terminal/iTerm. Windows: PowerShell 7+ или Git Bash, **не CMD**. Команды везде на двух строках вместо `&&`.

**Проект:**
```
mkdir team-mvp
cd team-mvp
git init
mkdir -p docs/research
```

Скопируйте в корень файлы из папки раздатки: `workshop.md` (плейбук), `prompts.md` (этот файл), по желанию `Product_Book_Template.md`. В `docs/research/` — все исследования и брифы команды.

**Служебные файлы (`.cursorrules`, `.env.local`) создавайте в Cursor** (File → New Text File → Cmd/Ctrl+S → имя с точкой, без расширения). Блокнот дописывает `.txt`, TextEdit сохраняет rtf — оба ломают проект.

**Cursor CLI:** Cmd/Ctrl+Shift+P → `Shell Command: Install 'cursor' command in PATH` → перезапустить терминал → `cursor .`

## ШПАРГАЛКИ

**Git-минимум дня:**
```
git status / git add . / git commit -m "..." / git push
git checkout -b feat/x        # новая ветка
git pull origin main          # перед merge
```

**Пять элементов хорошего промпта:**
```
[КОНТЕКСТ]     @product_book.md, какой слой, что уже сделано
[ЗАДАЧА]       одно предложение
[ВХОДНЫЕ]      какие § книги использовать
[ОГРАНИЧЕНИЯ]  чего не делать
[КРИТЕРИЙ]     как проверить, что готово
```
Без критерия Cursor не знает, когда «готово». С критерием — останавливается.
